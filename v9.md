# v9.0 — 消息过滤与路由

## 1. 版本目标

消费者可以按消息属性（header）过滤，只消费感兴趣的消息；支持基于规则的消息路由到不同 Topic。

**核心能力**：
- 消费者订阅时指定过滤条件，服务端过滤后返回
- 支持简单表达式（等值、范围、逻辑运算）
- 消息发送时根据路由规则自动转发到目标 Topic
- 减少网络传输和客户端处理开销

## 2. 核心功能说明

### 消息过滤

```
传统消费（无过滤）

Consumer 订阅 topic-A
      │
      ▼
recv() 拿到所有消息
      │
      ▼
客户端自己判断是否需要
      │
      ▼
不需要的消息也传输了，浪费带宽


过滤消费（v9.0）

Consumer 订阅 topic-A，filter="type=order AND amount>100"
      │
      ▼
recv() 服务端先过滤
      │
      ▼
只返回符合条件的消息
      │
      ▼
减少网络传输，客户端直接处理
```

### 过滤表达式语法

```
支持的运算符：
  =   等于         type=order
  !=  不等于       status!=cancelled
  >   大于         amount>100
  >=  大于等于     age>=18
  <   小于         price<50
  <=  小于等于     score<=60
  AND 逻辑与       type=order AND amount>100
  OR  逻辑或       status=pending OR status=processing
  ()  分组         (type=order OR type=refund) AND amount>100

示例：
  type=order                          # 只要订单消息
  amount>100 AND currency=USD         # 金额大于100美元的消息
  priority=high OR priority=urgent    # 高优先级或紧急消息
  region=cn AND (type=order OR type=payment)  # 中国区的订单或支付消息
```

### 消息路由

```
路由规则配置（服务端）

routes:
  - name: high-value-orders
    source: orders
    target: orders.high-value
    condition: amount>1000

  - name: cn-region
    source: events
    target: events.cn
    condition: region=cn

  - name: error-logs
    source: logs
    target: logs.error
    condition: level=ERROR


路由流程

Producer.send(topic="orders", msg={amount:1500, ...})
      │
      ▼
MQServer 检查路由规则
      │
      ▼
匹配到 high-value-orders 规则
      │
      ▼
写入 orders（原 Topic）
      │
      ▼
同时写入 orders.high-value（目标 Topic）
      │
      ▼
返回成功
```

## 4. 代码实现说明

### FilterExpression.java

```java
@Data
@AllArgsConstructor
public class FilterExpression {
    private String raw;           // 原始表达式字符串
    private ExprNode root;        // 解析后的 AST 根节点

    // AST 节点
    interface ExprNode {
        boolean evaluate(Map<String, String> headers);
    }

    // 比较节点：amount>100
    @Data
    static class CompareNode implements ExprNode {
        private String field;
        private String operator;  // =, !=, >, >=, <, <=
        private String value;

        @Override
        public boolean evaluate(Map<String, String> headers) {
            String actual = headers.get(field);
            if (actual == null) return false;

            switch (operator) {
                case "=":  return actual.equals(value);
                case "!=": return !actual.equals(value);
                case ">":  return Double.parseDouble(actual) > Double.parseDouble(value);
                case ">=": return Double.parseDouble(actual) >= Double.parseDouble(value);
                case "<":  return Double.parseDouble(actual) < Double.parseDouble(value);
                case "<=": return Double.parseDouble(actual) <= Double.parseDouble(value);
                default:   return false;
            }
        }
    }

    // 逻辑节点：AND / OR
    @Data
    static class LogicNode implements ExprNode {
        private String operator;  // AND, OR
        private ExprNode left;
        private ExprNode right;

        @Override
        public boolean evaluate(Map<String, String> headers) {
            if ("AND".equals(operator)) {
                return left.evaluate(headers) && right.evaluate(headers);
            } else if ("OR".equals(operator)) {
                return left.evaluate(headers) || right.evaluate(headers);
            }
            return false;
        }
    }
}
```

### ExpressionParser.java

```java
@Component
public class ExpressionParser {

    public FilterExpression parse(String expr) {
        if (expr == null || expr.trim().isEmpty()) {
            return null;
        }

        ExprNode root = parseLogic(expr);
        return new FilterExpression(expr, root);
    }

    private ExprNode parseLogic(String expr) {
        // 简化实现：先找 OR，再找 AND
        int orPos = findOperator(expr, "OR");
        if (orPos > 0) {
            String left = expr.substring(0, orPos).trim();
            String right = expr.substring(orPos + 2).trim();
            return new LogicNode("OR", parseLogic(left), parseLogic(right));
        }

        int andPos = findOperator(expr, "AND");
        if (andPos > 0) {
            String left = expr.substring(0, andPos).trim();
            String right = expr.substring(andPos + 3).trim();
            return new LogicNode("AND", parseLogic(left), parseLogic(right));
        }

        // 叶子节点：比较表达式
        return parseCompare(expr);
    }

    private ExprNode parseCompare(String expr) {
        // 匹配 field op value，如 amount>100
        for (String op : Arrays.asList(">=", "<=", "!=", "=", ">", "<")) {
            int pos = expr.indexOf(op);
            if (pos > 0) {
                String field = expr.substring(0, pos).trim();
                String value = expr.substring(pos + op.length()).trim();
                return new CompareNode(field, op, value);
            }
        }
        throw new IllegalArgumentException("Invalid expression: " + expr);
    }

    private int findOperator(String expr, String op) {
        // 跳过括号内的内容
        int level = 0;
        for (int i = 0; i < expr.length(); i++) {
            char c = expr.charAt(i);
            if (c == '(') level++;
            else if (c == ')') level--;
            else if (level == 0 && expr.startsWith(op, i)) {
                return i;
            }
        }
        return -1;
    }
}
```

### FilterEngine.java

```java
@Component
public class FilterEngine {

    @Autowired
    private ExpressionParser parser;

    public boolean match(Message<?> message, String filterExpr) {
        if (filterExpr == null || filterExpr.isEmpty()) {
            return true;  // 无过滤条件，全部匹配
        }

        FilterExpression expr = parser.parse(filterExpr);
        return expr.getRoot().evaluate(message.getHeaders());
    }
}
```

### RouteRuleManager.java

```java
@Component
public class RouteRuleManager {

    // topic → List<RouteRule>
    private final Map<String, List<RouteRule>> rules = new ConcurrentHashMap<>();

    @PostConstruct
    public void init() {
        // 从配置文件加载路由规则（简化示例）
        loadRules();
    }

    private void loadRules() {
        // 示例规则
        addRule(new RouteRule(
            "high-value-orders",
            "orders",
            "orders.high-value",
            "amount>1000",
            true  // 复制模式
        ));

        addRule(new RouteRule(
            "error-logs",
            "logs",
            "logs.error",
            "level=ERROR",
            true
        ));
    }

    public void addRule(RouteRule rule) {
        rules.computeIfAbsent(rule.getSourceTopic(), k -> new ArrayList<>())
             .add(rule);
    }

    public List<RouteRule> getRules(String topic) {
        return rules.getOrDefault(topic, Collections.emptyList());
    }
}
```

### MessageRouter.java

```java
@Component
public class MessageRouter {

    @Autowired
    private RouteRuleManager ruleManager;

    @Autowired
    private FilterEngine filterEngine;

    @Autowired
    private MQServer mqServer;

    public void route(String sourceTopic, Message<?> message) {
        List<RouteRule> rules = ruleManager.getRules(sourceTopic);

        for (RouteRule rule : rules) {
            if (filterEngine.match(message, rule.getCondition())) {
                // 匹配成功，写入目标 Topic
                mqServer.send(rule.getTargetTopic(), 0, message);

                if (!rule.isCopyMode()) {
                    // 移动模式，不写入源 Topic
                    return;
                }
            }
        }
    }
}
```

### MessageQueue.java（增加过滤逻辑）

```java
public class MessageQueue {

    @Autowired
    private FilterEngine filterEngine;

    public Message<?> recv(String consumerId) {
        Subscription sub = subscriptions.get(consumerId);
        if (sub == null) return null;

        int offset = sub.getOffset();
        String filter = sub.getFilter();  // 新增字段

        // 循环查找符合过滤条件的消息
        while (offset < store.size()) {
            Message<?> msg = store.read(offset);

            // 应用过滤
            if (filterEngine.match(msg, filter)) {
                msg.getHeaders().put("X-offset", String.valueOf(offset));
                return msg;
            }

            // 不匹配，自动跳过
            offset++;
            sub.setOffset(offset);
        }

        return null;  // 无符合条件的消息
    }
}
```

### MQServer.java（改动）

```java
@RestController
@RequestMapping("/mamq")
public class MQServer {

    @Autowired
    private MessageRouter router;

    // sub 接口增加 filter 参数
    @GetMapping("/sub")
    public Result<String> subscribe(@RequestParam String t,
                                    @RequestParam String cid,
                                    @RequestParam(required = false) String gid,
                                    @RequestParam(required = false) String filter) {
        MessageQueue mq = getOrCreate(t);
        Subscription sub = new Subscription(cid, gid, 0, filter);
        mq.subscribe(sub);
        return Result.ok("OK");
    }

    // send 接口触发路由
    @PostMapping("/send")
    public Result<Integer> send(@RequestParam String t,
                                @RequestParam(defaultValue = "0") int p,
                                @RequestBody Message message) {
        // 先写入源 Topic
        int offset = getOrCreate(t, p).send(message);

        // 触发路由
        router.route(t, message);

        return Result.ok(offset);
    }

    // 新增：管理路由规则
    @PostMapping("/admin/route/add")
    public Result<String> addRoute(@RequestBody RouteRule rule) {
        ruleManager.addRule(rule);
        return Result.ok("OK");
    }

    @GetMapping("/admin/route/list")
    public Result<List<RouteRule>> listRoutes(@RequestParam String t) {
        return Result.ok(ruleManager.getRules(t));
    }
}
```

## 6. 版本总结

### 学到了什么

v9.0 实现了消息的智能过滤和路由：

**服务端过滤**：消费者订阅时指定条件，服务端只返回符合条件的消息，减少网络传输。

**表达式引擎**：支持比较运算（=, >, <）和逻辑运算（AND, OR），解析为 AST 求值。

**消息路由**：根据规则自动将消息复制或移动到目标 Topic，实现消息分流。

**应用场景**：
- 多租户系统：按 tenantId 过滤，每个租户只消费自己的消息
- 日志分级：ERROR 日志路由到告警 Topic，INFO 日志保留在原 Topic
- 地域分流：按 region 字段路由到不同区域的 Topic

```
过滤流程

Consumer.sub(topic, filter="type=order AND amount>100")
      │
      ▼
MessageQueue.recv()
      │
      ▼
循环读取消息，应用过滤表达式
      │
      ▼
不匹配的消息自动跳过（offset 推进）
      │
      ▼
返回第一条匹配的消息


路由流程

Producer.send(topic="orders", msg={amount:1500})
      │
      ▼
写入 orders Topic
      │
      ▼
MessageRouter 检查规则
      │
      ▼
匹配 amount>1000 规则
      │
      ▼
复制到 orders.high-value Topic
```

### 与 v8.0 的对比

| 方面 | v8.0 | v9.0 |
|------|------|------|
| 消费方式 | 拿到所有消息 | 服务端过滤后返回 |
| 网络开销 | 传输所有消息 | 只传输匹配消息 |
| 客户端逻辑 | 需自己判断过滤 | 直接处理，无需判断 |
| 消息分流 | 手动发送到不同 Topic | 自动路由 |
| 多租户支持 | 需业务层实现 | 过滤器原生支持 |

### 这个版本的局限

| 问题 | 现象 | 未来解决 |
|------|------|----------|
| 表达式功能有限 | 不支持正则、IN、LIKE | 扩展运算符 |
| 过滤性能 | 每条消息都要求值 | 索引加速（Bloom Filter） |
| 路由规则静态 | 需重启生效 | 动态热加载 |
| 无管理界面 | 只能 API 操作 | v10.0 Web UI |

### 设计要点

- **AST 求值**：表达式解析为抽象语法树，递归求值，支持嵌套逻辑
- **自动跳过**：不匹配的消息自动推进 offset，对消费者透明
- **复制 vs 移动**：copyMode=true 时源和目标都写，false 时只写目标
- **路由时机**：在 send 接口写入后立即触发，保证实时性
- **过滤缓存**：FilterExpression 解析后缓存，避免重复解析

### 性能优化建议

- **索引过滤**：对高频过滤字段（如 type, region）建立索引，跳过不匹配的 segment
- **Bloom Filter**：快速判断消息是否可能匹配，减少反序列化开销
- **批量过滤**：一次读取多条消息，批量应用过滤，减少 I/O 次数
- **规则编译**：将表达式编译为字节码，提升求值速度
