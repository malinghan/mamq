# v4.0 — 存储分段

## 1. 版本目标

突破 v2.0 单文件存储上限，支持任意消息量写入。同时持久化索引文件和消费位点，重启后消息和消费进度都能完整恢复。

**只改存储层**：HTTP 接口、客户端、订阅逻辑与 v3.0 完全一致。

---

## 2. 核心功能说明

### 为什么需要分段

v2.0 的 `Store` 在初始化时预分配了固定大小的 `MappedByteBuffer`（如 10MB），写满后无法继续写入。v4.0 改为多个 segment 文件，每个 segment 固定大小，写满后自动创建下一个。

### 三个新概念

**Segment（分段文件）**：每个 segment 对应一个 `.dat` 文件，命名为 `{topic}.0.dat`、`{topic}.1.dat`...，固定大小（如 1GB）。

**索引文件 `.idx`**：记录每条消息的位置信息 `{globalOffset},{segmentId},{localPosition},{length}`，一行一条，启动时直接加载，无需扫描 `.dat`。

**位点文件 `.sub`**：记录每个消费者的消费进度 `{consumerId},{offset}`，ACK 时同步写入，重启后恢复。

### 存储结构

```
mamq-data/
  {topic}.0.dat       ← segment 0，写满后不再写入
  {topic}.1.dat       ← segment 1，当前写入
  {topic}.idx         ← 全局索引文件
  {topic}.sub         ← 消费位点文件
```

### Indexer 升级

```
v2.0 Indexer：
  offset → Entry{ position, length }        ← 单文件，position 是文件内字节位置

v4.0 Indexer：
  offset → Entry{ segmentId, localPosition, length }  ← 多文件，需要先找到哪个 segment
```

### 读写流程

```
写入流程

store.write(msg)
      │
      ▼
  当前 segment 还有空间？
  ┌───┴───┐
  │ Yes   │ No → 创建新 segment（segmentId++）
  ▼
  写入当前 segment（localPosition = segment.writePos）
      │
      ▼
  Indexer.addEntry(globalOffset, segmentId, localPosition, length)
      │
      ▼
  追加写入 {topic}.idx 文件（一行）

读取流程

store.read(globalOffset)
      │
      ▼
  Indexer.getEntry(globalOffset)
  → {segmentId, localPosition, length}
      │
      ▼
  segments.get(segmentId).read(localPosition, length)
      │
      ▼
  返回 Message

启动恢复流程

Store.init()
      │
      ▼
  {topic}.idx 存在？
  ┌───┴───┐
  │ Yes   │ No → 扫描所有 .dat 重建索引并写 .idx
  ▼
  逐行读取 .idx，重建内存 Indexer
      │
      ▼
  读取 {topic}.sub，恢复各消费者 offset

ACK 时持久化位点

ack(consumerId, offset)
      │
      ▼
  sub.setOffset(offset + 1)
      │
      ▼
  将所有 subscriptions 写入 {topic}.sub 文件（覆盖写）
```

### 层次关系

```
MessageQueue
    ├── subscriptions（启动时从 .sub 恢复，ack 时写回 .sub）
    └── Store
          ├── segments: List<Segment>
          │     Segment.0 → {topic}.0.dat（MappedByteBuffer）
          │     Segment.1 → {topic}.1.dat（MappedByteBuffer，当前写入）
          └── Indexer
                └── Map<globalOffset, Entry{segmentId, localPos, length}>
                      启动时从 .idx 加载，写入时追加到 .idx
```

## 4. 代码实现说明

### Segment.java

```java
public class Segment {
    private static final int HEADER_LEN = 10;

    private final int segmentId;
    private final MappedByteBuffer buffer;
    private int writePosition = 0;
    private final int capacity;

    public Segment(int segmentId, File file, int capacity) throws IOException {
        this.segmentId = segmentId;
        this.capacity = capacity;
        RandomAccessFile raf = new RandomAccessFile(file, "rw");
        raf.setLength(capacity);
        this.buffer = raf.getChannel().map(FileChannel.MapMode.READ_WRITE, 0, capacity);
    }

    public boolean hasSpace(int dataLen) {
        return writePosition + HEADER_LEN + dataLen <= capacity;
    }

    // 写入消息体，返回 localPosition（消息体起始位置，不含头）
    public int write(byte[] data) {
        String header = String.format("%010d", data.length);
        buffer.position(writePosition);
        buffer.put(header.getBytes(StandardCharsets.UTF_8));
        int localPosition = writePosition + HEADER_LEN;
        buffer.put(data);
        writePosition = buffer.position();
        return localPosition;
    }

    public byte[] read(int localPosition, int length) {
        byte[] data = new byte[length];
        buffer.position(localPosition);
        buffer.get(data);
        return data;
    }

    public int getSegmentId() { return segmentId; }
    public int getWritePosition() { return writePosition; }
}
```

### Indexer.java（升级）

```java
public class Indexer {

    @Data
    @AllArgsConstructor
    public static class Entry {
        private int segmentId;
        private int localPosition;
        private int length;
    }

    private final Map<Integer, Entry> index = new HashMap<>();
    private int nextOffset = 0;

    public int addEntry(int segmentId, int localPosition, int length) {
        index.put(nextOffset, new Entry(segmentId, localPosition, length));
        return nextOffset++;
    }

    public Entry getEntry(int offset) {
        return index.get(offset);
    }

    public int size() { return nextOffset; }
}
```

### Store.java（核心改动）

```java
public class Store {
    private static final int SEGMENT_SIZE = 1024 * 1024 * 10; // 10MB，演示用

    private final String topic;
    private final File dataDir;
    private final List<Segment> segments = new ArrayList<>();
    private final Indexer indexer = new Indexer();
    private final File idxFile;
    private PrintWriter idxWriter;

    public Store(String topic) {
        this.topic = topic;
        this.dataDir = new File("mamq-data");
        this.dataDir.mkdirs();
        this.idxFile = new File(dataDir, topic + ".idx");
        init();
    }

    private void init() {
        if (idxFile.exists()) {
            loadFromIdx();   // 优先从 .idx 加载，快
        } else {
            scanDatFiles();  // 降级：扫描 .dat 重建
        }
        // 打开 .idx 追加写
        try {
            idxWriter = new PrintWriter(new FileWriter(idxFile, true));
        } catch (IOException e) {
            throw new RuntimeException(e);
        }
        // 确保至少有一个可写 segment
        if (segments.isEmpty() || !currentSegment().hasSpace(1)) {
            openNewSegment();
        }
    }

    private void loadFromIdx() {
        try (BufferedReader br = new BufferedReader(new FileReader(idxFile))) {
            String line;
            while ((line = br.readLine()) != null) {
                // 格式：globalOffset,segmentId,localPosition,length
                String[] parts = line.split(",");
                int globalOffset  = Integer.parseInt(parts[0]);
                int segmentId     = Integer.parseInt(parts[1]);
                int localPosition = Integer.parseInt(parts[2]);
                int length        = Integer.parseInt(parts[3]);
                ensureSegment(segmentId);
                indexer.addEntry(segmentId, localPosition, length);
            }
        } catch (IOException e) {
            throw new RuntimeException(e);
        }
    }

    private void scanDatFiles() {
        // 扫描 topic.0.dat, topic.1.dat ... 重建索引
        int segId = 0;
        while (true) {
            File f = new File(dataDir, topic + "." + segId + ".dat");
            if (!f.exists()) break;
            Segment seg = openSegment(segId, f);
            // 扫描 segment 内容重建 indexer（逻辑同 v2.0 init）
            scanSegment(seg);
            segId++;
        }
    }

    public synchronized int write(Message message) {
        String json = JSON.toJSONString(message);
        byte[] data = json.getBytes(StandardCharsets.UTF_8);

        if (!currentSegment().hasSpace(data.length)) {
            openNewSegment();
        }

        Segment seg = currentSegment();
        int localPosition = seg.write(data);
        int globalOffset = indexer.addEntry(seg.getSegmentId(), localPosition, data.length);

        // 追加写入 .idx
        idxWriter.println(globalOffset + "," + seg.getSegmentId()
                + "," + localPosition + "," + data.length);
        idxWriter.flush();

        return globalOffset;
    }

    public Message read(int globalOffset) {
        Indexer.Entry entry = indexer.getEntry(globalOffset);
        if (entry == null) return null;
        Segment seg = segments.get(entry.getSegmentId());
        byte[] data = seg.read(entry.getLocalPosition(), entry.getLength());
        return JSON.parseObject(new String(data, StandardCharsets.UTF_8), Message.class);
    }

    public int size() { return indexer.size(); }

    private Segment currentSegment() { return segments.get(segments.size() - 1); }

    private void openNewSegment() {
        int segId = segments.size();
        File f = new File(dataDir, topic + "." + segId + ".dat");
        segments.add(openSegment(segId, f));
    }

    private Segment openSegment(int segId, File f) {
        try {
            return new Segment(segId, f, SEGMENT_SIZE);
        } catch (IOException e) {
            throw new RuntimeException(e);
        }
    }

    private void ensureSegment(int segmentId) {
        while (segments.size() <= segmentId) {
            int id = segments.size();
            File f = new File(dataDir, topic + "." + id + ".dat");
            segments.add(openSegment(id, f));
        }
    }

    private void scanSegment(Segment seg) { /* 同 v2.0 init 扫描逻辑 */ }
}
```

### MessageQueue.java（位点持久化）

```java
public class MessageQueue {
    private final String topic;
    private final Store store;
    private final Map<String, Subscription> subscriptions = new ConcurrentHashMap<>();
    private final File subFile;

    public MessageQueue(String topic) {
        this.topic = topic;
        this.store = new Store(topic);
        this.subFile = new File("mamq-data", topic + ".sub");
        loadSubscriptions();  // 启动时从 .sub 恢复
    }

    // 从 .sub 文件恢复消费位点
    private void loadSubscriptions() {
        if (!subFile.exists()) return;
        try (BufferedReader br = new BufferedReader(new FileReader(subFile))) {
            String line;
            while ((line = br.readLine()) != null) {
                // 格式：consumerId,offset
                String[] parts = line.split(",");
                String cid = parts[0];
                int offset = Integer.parseInt(parts[1]);
                subscriptions.put(cid, new Subscription(topic, cid, offset));
            }
        } catch (IOException e) {
            throw new RuntimeException(e);
        }
    }

    // 持久化所有消费位点到 .sub 文件
    private synchronized void persistSubscriptions() {
        try (PrintWriter pw = new PrintWriter(new FileWriter(subFile, false))) {
            subscriptions.forEach((cid, sub) ->
                pw.println(cid + "," + sub.getOffset()));
        } catch (IOException e) {
            throw new RuntimeException(e);
        }
    }

    public int ack(String consumerId, int offset) {
        Subscription sub = subscriptions.get(consumerId);
        if (sub != null) {
            sub.setOffset(offset + 1);
            persistSubscriptions();  // ACK 时同步持久化
        }
        return offset;
    }

    // send / recv / batch / subscribe / unsubscribe / stat 与 v3.0 一致，略
}
```

## 6. 版本总结

### 学到了什么

v4.0 解决了存储的两个核心问题：

**分段存储**：把一个大文件拆成多个固定大小的 segment，写满自动滚动，理论上支持无限写入。读取时通过 Indexer 透明路由到正确的 segment，调用方无感知。

**位点持久化**：ACK 时同步写入 `.sub` 文件，重启后直接恢复，消费者不需要从头消费。

```
写入：msg → 当前 segment → 追加 .idx 一行
读取：globalOffset → .idx → segmentId + localPos → segment.read()
恢复：启动时读 .idx 重建 Indexer，读 .sub 恢复 offset
```

### 与 v2.0/v3.0 的对比

| 方面 | v2.0/v3.0 | v4.0 |
|------|-----------|------|
| 存储文件 | 单个 `.dat`，固定上限 | 多个 `.dat`，自动分段 |
| 索引 | 内存，重启扫描重建 | 持久化到 `.idx`，重启直接加载 |
| 消费位点 | 内存，重启归零 | 持久化到 `.sub`，重启恢复 |
| 启动速度 | 需扫描全量 `.dat` | 直接读 `.idx`，O(n) 行数 |

### 这个版本的局限

| 问题 | 现象 | 下一版解决 |
|------|------|-----------|
| 多消费者重复消费 | 每个 cid 独立 offset，同一消息被多人消费 | v5.0 消费者组 |
| ACK 每次全量写 .sub | 消费者多时写放大 | 可优化为增量追加，当前够用 |
| 无消费失败处理 | 回调异常消息丢失 | v6.0 重试 + DLQ |
