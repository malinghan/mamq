# v3.0 — 客户端封装，支持 push 模式

## 1. 版本目标

在 v2.0 的 HTTP 接口之上封装 Java 客户端，让调用方不用手写 HTTP 请求，直接用对象操作。

新增 push 模式：`MABroker` 后台线程自动轮询，消息到达时回调用户注册的 `MAListener`，用户无需关心轮询细节。

**服务端代码完全不动**，只新增 `client/` 包。

---

## 2. 核心功能说明

### 两种消费模式

**Pull 模式**：用户主动调用 `consumer.recv()`，拿到消息后手动 `ack`。适合需要精确控制消费节奏的场景。

**Push 模式**：用户注册 `MAListener`，`MABroker` 后台线程每 100ms 轮询一次，有消息时自动回调 `listener.onMessage(msg)`，回调完成后自动 `ack`。适合事件驱动场景。

### 客户端组件职责

```
MABroker
  ├── 管理 Producer/Consumer 的创建和生命周期
  ├── 维护服务端地址（host:port）
  └── 后台线程池，驱动所有 push 模式的 Consumer 轮询

MAProducer
  └── send(topic, message) → HTTP POST /mamq/send

MAConsumer
  ├── sub(topic)           → HTTP GET /mamq/sub
  ├── recv(topic)          → HTTP GET /mamq/recv
  ├── ack(topic, offset)   → HTTP GET /mamq/ack
  ├── stat(topic)          → HTTP GET /mamq/stat
  └── listen(topic, listener) → 注册回调，交给 MABroker 驱动

MAListener（接口）
  └── onMessage(Message msg) → 用户实现的消息处理逻辑
```

### Push 模式工作原理

```
MABroker 后台线程（每个 listen 注册一个线程）
    │
    ▼
循环：consumer.recv(topic)
    │
    ▼
msg != null ?
┌───┴───┐
│ Yes   │ No → Thread.sleep(100ms) → 继续循环
▼
listener.onMessage(msg)   ← 用户回调
    │
    ▼
consumer.ack(topic, msg.offset)  ← 自动 ACK
    │
    ▼
继续循环（不 sleep，立即取下一条）
```

## 4. 代码实现说明

### MAListener.java

```java
@FunctionalInterface
public interface MAListener<T> {
    void onMessage(Message<T> message);
}
```

函数式接口，支持 lambda 写法。

### MAProducer.java

```java
public class MAProducer {

    private final String baseUrl;

    public MAProducer(String baseUrl) {
        this.baseUrl = baseUrl;
    }

    public int send(String topic, Message<?> message) {
        String url = baseUrl + "/mamq/send?t=" + topic;
        String body = JSON.toJSONString(message);
        String response = HttpUtil.post(url, body);  // 简单 HTTP 工具类
        Result<Integer> result = JSON.parseObject(response,
            new TypeReference<Result<Integer>>() {});
        return result.getData();
    }
}
```

### MAConsumer.java

```java
public class MAConsumer<T> {

    private final String baseUrl;
    private final String topic;
    private final String consumerId;
    private final Class<T> bodyType;

    public MAConsumer(String baseUrl, String topic,
                      String consumerId, Class<T> bodyType) {
        this.baseUrl = baseUrl;
        this.topic = topic;
        this.consumerId = consumerId;
    }

    public void sub() {
        String url = baseUrl + "/mamq/sub?t=" + topic + "&cid=" + consumerId;
        HttpUtil.get(url);
    }

    // 拉取一条消息（不推进 offset）
    public Message<T> recv() {
        String url = baseUrl + "/mamq/recv?t=" + topic + "&cid=" + consumerId;
        String response = HttpUtil.get(url);
        Result<Message<T>> result = JSON.parseObject(response,
            new TypeReference<Result<Message<T>>>() {});
        return result.getData();
    }

    public int ack(int offset) {
        String url = baseUrl + "/mamq/ack?t=" + topic
                   + "&cid=" + consumerId + "&offset=" + offset;
        String response = HttpUtil.get(url);
        Result<Integer> result = JSON.parseObject(response,
            new TypeReference<Result<Integer>>() {});
        return result.getData();
    }

    public Stat stat() {
        String url = baseUrl + "/mamq/stat?t=" + topic + "&cid=" + consumerId;
        String response = HttpUtil.get(url);
        Result<Stat> result = JSON.parseObject(response,
            new TypeReference<Result<Stat>>() {});
        return result.getData();
    }

    // 注册 push 监听，返回 this 供链式调用
    // 实际轮询由 MABroker 驱动，这里只是注册
    public MAConsumer<T> listen(MAListener<T> listener) {
        sub();  // 自动订阅
        // MABroker 会扫描已注册的 listener 并启动轮询线程
        this.listener = listener;
        return this;
    }

    // package-private，供 MABroker 访问
    MAListener<T> listener;
}
```

### MABroker.java

```java
public class MABroker {

    private static final MABroker DEFAULT = new MABroker("http://localhost:8765");

    private final String baseUrl;
    private final ExecutorService executor = Executors.newCachedThreadPool();
    // 记录所有注册了 listener 的 consumer，后台线程轮询
    private final List<MAConsumer<?>> consumers = new CopyOnWriteArrayList<>();

    private MABroker(String baseUrl) {
        this.baseUrl = baseUrl;
    }

    public static MABroker getDefault() {
        return DEFAULT;
    }

    public static MABroker of(String host, int port) {
        return new MABroker("http://" + host + ":" + port);
    }

    public MAProducer createProducer() {
        return new MAProducer(baseUrl);
    }

    public <T> MAConsumer<T> createConsumer(String topic, String consumerId) {
        MAConsumer<T> consumer = new MAConsumer<>(baseUrl, topic, consumerId, null);
        consumers.add(consumer);
        return consumer;
    }

    // 启动后台轮询（在 listen() 注册后调用）
    public void startPolling(MAConsumer<?> consumer) {
        executor.submit(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                try {
                    pollOnce(consumer);
                } catch (Exception e) {
                    // 轮询异常不中断循环
                }
            }
        });
    }

    @SuppressWarnings("unchecked")
    private <T> void pollOnce(MAConsumer<T> consumer) throws InterruptedException {
        Message<T> msg = consumer.recv();
        if (msg == null) {
            Thread.sleep(100);  // 无消息时等待 100ms
            return;
        }
        if (consumer.listener != null) {
            consumer.listener.onMessage(msg);
            consumer.ack(msg.getOffset());  // 回调成功后自动 ACK
        }
        // 有消息时不 sleep，立即取下一条
    }

    public void shutdown() {
        executor.shutdownNow();
    }
}
```

> 注意：`Message` 需要增加 `offset` 字段，由服务端在 recv 响应时填入，方便客户端 ack。

### MAMqDemo.java

```java
public class MAMqDemo {

    public static void main(String[] args) throws Exception {
        MABroker broker = MABroker.getDefault();
        MAProducer producer = broker.createProducer();
        MAConsumer<String> consumer = broker.createConsumer("demo.topic", "demo-c1");
        consumer.sub();

        Scanner scanner = new Scanner(System.in);
        System.out.println("命令: p=生产 c=消费 s=统计 b=批量生产10条 l=启动push监听 q=退出");

        while (scanner.hasNextLine()) {
            String cmd = scanner.nextLine().trim();
            switch (cmd) {
                case "p" -> {
                    long id = System.currentTimeMillis();
                    int offset = producer.send("demo.topic",
                        new Message<>(id, "msg-" + id, null));
                    System.out.println("sent, offset=" + offset);
                }
                case "c" -> {
                    Message<String> msg = consumer.recv();
                    if (msg == null) {
                        System.out.println("no message");
                    } else {
                        System.out.println("recv: " + msg);
                        consumer.ack(msg.getOffset());
                    }
                }
                case "s" -> System.out.println("stat: " + consumer.stat());
                case "b" -> {
                    for (int i = 0; i < 10; i++) {
                        producer.send("demo.topic",
                            new Message<>((long) i, "batch-" + i, null));
                    }
                    System.out.println("sent 10 messages");
                }
                case "l" -> {
                    consumer.listen(msg -> System.out.println("push recv: " + msg.getBody()));
                    broker.startPolling(consumer);
                    System.out.println("push listener started");
                }
                case "q" -> {
                    broker.shutdown();
                    System.exit(0);
                }
            }
        }
    }
}
```

## 6. 版本总结

### 学到了什么

v3.0 展示了 MQ 客户端的两种经典设计模式：

**Pull 模式**：消费者主动拉取，控制权在消费者手里。适合批处理、限速消费等场景。

**Push 模式**：Broker 主动推送（通过轮询模拟），消费者只需关注业务逻辑。适合实时处理、事件驱动场景。

```
Pull：用户调用 consumer.recv() → 拿消息 → 处理 → consumer.ack()
Push：注册 listener → MABroker 后台轮询 → 有消息自动回调 → 自动 ack
```

### 设计要点

- `MABroker` 是客户端的核心，管理连接配置和后台线程，用户不需要关心 HTTP 细节
- `MAListener` 设计为函数式接口，支持 lambda，降低使用门槛
- push 模式下 ACK 由 `MABroker` 自动完成，用户只需关注 `onMessage` 里的业务逻辑
- 无消息时 sleep 100ms，避免空轮询浪费 CPU；有消息时立即取下一条，保证吞吐

### 与前两个版本的对比

| 方面 | v1.0 | v2.0 | v3.0 |
|------|------|------|------|
| 调用方式 | 手写 curl | 手写 curl | Java 对象调用 |
| 消费模式 | 仅 pull | 仅 pull | pull + push |
| ACK 方式 | 手动 | 手动 | pull 手动 / push 自动 |
| 服务端变化 | — | 加持久化 | 不变 |

### 这个版本的局限

| 问题 | 现象 | 下一版解决 |
|------|------|-----------|
| 单文件存储上限 | 消息量大时文件写满 | v4.0 分段存储 |
| 消费位点不持久化 | 重启后 offset 归零 | v4.0 `.sub` 文件 |
| 轮询固定 100ms | 空轮询浪费资源 | v7.0 自适应退避 |
| push 无重试 | 回调异常消息丢失 | v6.0 重试 + DLQ |
